# Отчет о проделанной работе: пакет валидации симуляционного стенда роя автономных агентов

## Цель работы

Подготовить чистый, воспроизводимый комплект документации и отчетных материалов для ROS 2-ориентированного симуляционного стенда роя автономных агентов. Комплект описывает порядок запуска стенда, взаимодействие модулей, собираемые метрики и результаты опубликованного диагностического набора экспериментов.

## Выполненные работы

1. Проанализирована структура репозитория и отделено описание симуляционного стенда от академической терминологии.
2. Нормализован язык документации: проект представлен как инженерный симуляционный стенд.
3. Подготовлен обновленный README с локальным запуском и запуском в контейнере.
4. Сформирован комплект таблиц со сводными результатами диагностической матрицы.
5. Подготовлен комплект графиков для визуального представления распределения запусков и коэффициентов успешности.
6. Подготовлено UML/data-flow описание симуляционного пайплайна.

## Матрица экспериментов

Опубликованная матрица содержит шесть сценариев, четыре архитектуры и тридцать seed-значений для каждой пары «сценарий / архитектура»:

```text
6 x 4 x 30 = 720 запусков
```

Классы сценариев:

- S1: штатная навигация.
- S2: среда с высокой плотностью препятствий.
- S3: деградация связи.
- S4: частичный отказ агентов.
- S5: вычислительная деградация.
- S6: комбинированный стресс.

Архитектуры:

- Central-A*.
- Реактивное управление.
- Децентрализованное управление на основе правил.
- Dec-POMDP + эвристическая коррекция.

### 6. Тепловая карта успешности «сценарий / архитектура»

![Тепловая карта успешности](../figures/fig_success_rate_heatmap.png)

Тепловая карта показывает, что единственная проблемная зона — это сочетание сценария S6 и реактивной архитектуры. Все остальные пары «сценарий / архитектура» демонстрируют полную успешность.

## Основные наблюдения

1. По коэффициенту успешности все архитектуры стабильны в сценариях S1-S5.
2. В сценарии S6 базовый реактивный алгоритм падает до 23.3% успешности, тогда как остальные архитектуры сохраняют 100%.
3. Диагностическая матрица является сбалансированной: по каждой архитектуре и по каждому сценарию выполнено одинаковое количество запусков.
4. Набор результатов является чистым с точки зрения тайм-аутов runner-а и некорректных диагностических строк.
5. Сценарий S6 является ключевым стресс-сценарием, поскольку объединяет препятствия, деградацию связи, отказы и вычислительную деградацию.

## Инженерная интерпретация

Текущий комплект следует рассматривать как детерминированный бенчмарк для проверки симуляционного пайплайна, контура сбора данных и сравнительного поведения архитектур управления. Он подходит для регрессионного тестирования и алгоритмического сравнения в headless-кинематическом режиме. Для утверждений, зависящих от реалистичной динамики, временных характеристик middleware, сенсоров и исполнительных механизмов, требуется отдельный этап физически более достоверной валидации.
